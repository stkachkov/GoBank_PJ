# GoBank: Банковский сервис на Go

## Описание проекта

**GoBank** — это REST API для банковского сервиса, реализованный на языке Go. Сервис предоставляет основной банковский функционал, включая управление пользователями, счетами, картами и кредитами.

Проект демонстрирует принципы чистой архитектуры (слои `models`, `repository`, `service`, `handler`), безопасную работу с данными (PGP, HMAC, bcrypt) и использование Docker для оркестрации. Вся конфигурация приложения управляется через переменные окружения.

## Основные возможности

*   **Управление пользователями:** Регистрация и аутентификация с использованием JWT.
*   **Банковские счета:** Создание счетов (только в RUB), пополнение, снятие и переводы между счетами.
*   **Банковские карты:** Выпуск виртуальных карт (с валидацией по алгоритму Луна), безопасное хранение данных (PGP + HMAC), верификация CVV.
*   **Кредитование:** Подача заявки на кредит, расчет аннуитетных платежей и просмотр графика платежей.
*   **Финансовая аналитика:** Получение статистики по доходам/расходам, анализ кредитной нагрузки и прогноз баланса.
*   **Интеграции:**
    *   Получение ключевой ставки ЦБ РФ через SOAP.
    *   Отправка email-уведомлений через SMTP.
*   **Фоновые задачи:** Автоматическое списание платежей по кредитам и начисление штрафов с помощью внутреннего планировщика (scheduler).

## Технологический стек

*   **Язык:** Go
*   **База данных:** PostgreSQL
*   **Оркестрация:** Docker & Docker Compose
*   **Маршрутизация:** `gorilla/mux`
*   **Аутентификация:** `golang-jwt/jwt/v5`
*   **Безопасность:** `golang.org/x/crypto` (bcrypt, PGP)
*   **Логирование:** `sirupsen/logrus`

## Запуск и использование

### Требования
*   **Docker**
*   **Docker Compose**

### Шаг 1: Настройка окружения

Вся конфигурация проекта задается через переменные окружения. Для локального запуска используется файл `.env`.

1.  **Создайте файл конфигурации:**
    В корневой директории проекта (`GoBank_PJ/`) создайте копию файла `.env.example` и назовите ее `.env`.

    ```bash
    cp .env.example .env
    ```

2.  **Заполните `.env` файл:**
    Откройте созданный файл `.env` и заполните переменные. Ниже приведено описание каждой из них.

    | Переменная                 | Описание                                                                   | Пример значения                              |
    |----------------------------|----------------------------------------------------------------------------|----------------------------------------------|
    | `POSTGRES_DB`              | Название базы данных PostgreSQL.                                           | `gobank_db`                                  |
    | `POSTGRES_USER`            | Имя пользователя для подключения к БД.                                     | `user`                                       |
    | `POSTGRES_PASSWORD`        | Пароль для подключения к БД.                                               | `password`                                   |
    | `SERVER_PORT`              | Порт, который слушает приложение **внутри** Docker-контейнера.             | `8080`                                       |
    | `BANK_SERVICE_PORT`        | Порт на **вашем компьютере**, который будет перенаправлен на `SERVER_PORT` контейнера. | `8080`                                       |
    | `JWT_SECRET`               | Длинный, случайный секретный ключ для подписи JWT-токенов.                 | `your_super_secret_jwt_key_...`              |
    | `PGP_PRIVATE_KEY`          | Приватный PGP-ключ, используется для шифрования данных карт.               | `"-----BEGIN PGP.........END PGP-----"`   |
    | `PGP_PRIVATE_KEY_PASSPHRASE` | Пароль от вашего PGP-ключа.                                                                                | `your_pgp_passphrase`                        |
    | `HMAC_SECRET_KEY`          | Длинный, случайный секретный ключ для HMAC-подписей (целостность данных).                                  | `your_super_secret_hmac_key_...`             |
    | `SMTP_HOST`                | Адрес вашего SMTP-сервера для отправки email.                                                              | `smtp.example.com`                           |
    | `SMTP_PORT`                | Порт SMTP-сервера.                                                                                         | `587`                                        |
    | `SMTP_USERNAME`            | Имя пользователя для аутентификации на SMTP-сервере.                                                       | `user@example.com`                           |
    | `SMTP_PASSWORD`            | Пароль для аутентификации на SMTP-сервере.                                                                 | `password`                                   |
    | `SMTP_FROM`                | Email-адрес, который будет указан в поле "От кого".                                                        | `noreply@example.com`                        |
    
    **Важно:** Для работы сервиса карт необходимо сгенерировать и вставить реальный PGP-ключ в `PGP_PRIVATE_KEY` и пароль от него в `PGP_PRIVATE_KEY_PASSPHRASE`.

### Шаг 2: Запуск проекта

1.  Откройте терминал в корневой директории `GoBank_PJ/`.
2.  Выполните команду для сборки и запуска контейнеров:
    ```bash
    docker-compose up --build
    ```
    Эта команда:
    *   Соберет Docker-образ для `bank-service`.
    *   Запустит контейнеры `bank-service` и `postgres-db`.
    *   Отобразит логи обоих сервисов в вашем терминале.

    Чтобы запустить сервисы в фоновом режиме, добавьте флаг `-d`: `docker-compose up --build -d`.

3.  Сервис будет доступен по адресу `http://localhost:{BANK_SERVICE_PORT}` (например, `http://localhost:8080`).

4.  Чтобы остановить все сервисы, выполните:
    ```bash
    docker-compose down
    ```

## Краткий справочник по API

### Публичные эндпоинты
*   `POST /register` - Регистрация нового пользователя.
*   `POST /login` - Аутентификация и получение JWT.
*   `GET /cbr/key-rate` - Получить ключевую ставку ЦБ РФ.

### Защищенные эндпоинты
Все последующие запросы требуют заголовок `Authorization: Bearer <ВАШ_JWT_ТОКЕН>`.

*   **Счета (`/accounts`):**
    *   `POST /accounts` - Создать счет.
    *   `GET /accounts` - Получить список счетов.
    *   `POST /accounts/{id}/deposit` - Пополнить счет.
    *   `POST /accounts/{id}/withdraw` - Снять средства.
    *   `POST /accounts/{id}/transfer` - Перевести средства.

*   **Карты (`/cards`):**
    *   `POST /cards` - Выпустить карту для счета.
    *   `GET /accounts/{id}/cards` - Получить карты счета.
    *   `POST /cards/{id}/verify-cvv` - Проверить CVV карты.

*   **Кредиты (`/credits`):**
    *   `POST /credits` - Подать заявку на кредит.
    *   `GET /credits` - Получить кредиты пользователя.
    *   `GET /credits/{id}/schedule` - График платежей по кредиту.

*   **Аналитика (`/analytics`):**
    *   `GET /analytics/income-expense?year=...&month=...` - Получить доходы/расходы за месяц.
    *   `GET /analytics/credit-load` - Получить аналитику по кредитной нагрузке.
    *   `GET /accounts/{id}/predict?days=...` - Спрогнозировать баланс счета.
